## 1. 产品概述

AI-Blockly-IDE 是一款**自然语言驱动的硬件开发桌面 IDE**，用户只需用自然语言描述产品需求，系统即可自动完成：

```
自然语言描述产品需求
        ↓
AI 自动分析并选型元器件（Analyst Agent）
        ↓
AI 生成元件清单 + 连线图 + 引脚定义（Architect Agent）
        ↓
AI 生成 Arduino/ESP32 代码（Programmer Agent）
        ↓
编译验证 + 导出 BOM / 原理图 / PCB 文件
```

**核心能力：**
- 支持 ESP32、ESP32-S3、Arduino UNO、Nano 等开发板
- 142+ 真实元器件产品图片渲染
- BOM 元件清单（嘉立创 JLCPCB 标准格式）
- KiCad 原理图导出（可直接用于嘉立创下单）
- 连线图 / 面包板视图一键切换

### 3.2 配置步骤

1. 点击右上角 **⚙ 模型配置**
2. 在左侧下拉框选择模型商
3. 中间下拉框选择具体模型
4. 在 **API Key** 输入框粘贴密钥
5. 点击 **💾 保存配置**
6. 关闭窗口后即可使用

> Key 使用 AES-256 加密存储在本地 `%APPDATA%\AI-Blockly-IDE\config.json`

## 4. AI 生成流程

### 4.1 标准流程（自然语言输入）

1. **在左侧底部输入框输入产品描述**
   ```
   例如：帮我设计一个聊天机器人，需要实现实时语音交互
   ```
2. **点击 ▶ 发送**，系统自动执行三阶段：

   **Stage 1 - 硬件分析师（Analyst）**
   - 分析产品需求
   - 根据 HARDWARE_KNOWLEDGE 选型规则选择元器件
   - 输出：元件清单（components）

   **Stage 2 - 架构设计师（Architect）**
   - 规划引脚连接
   - 分配 GPIO 引脚资源
   - 输出：连接定义（connections）+ 连线图坐标

   **Stage 3 - 程序员（Programmer）**
   - 生成 Arduino/ESP32 代码
   - 包含引脚定义、库引用、主逻辑、串口通信等
   - 输出：arduinoCode

3. **完成后**，中间画布显示元器件和连线，右侧面板显示代码和元件清单

### 4.2 硬件选型参考

**语音聊天机器人 →**
ESP32 + INMP441/MAX9814（麦克风）+ UDA1334/PAM8403（音频输出）+ OLED（状态显示）+ WiFi（连接云端 LLM）

**语音助手（带屏幕）→**
ESP32-S3 + 数字麦克风 + TFT 彩屏 + 扬声器

**智能家居控制 →**
ESP32 + 语音模块 + 继电器 + 传感器

**环境监测站 →**
ESP32 + 多种传感器（温湿度/气体/光敏）+ LCD/OLED

## 5. 导入功能

点击顶部 **📥 导入** 按钮，出现下拉菜单：

### 5.1 导入产品描述文档

支持格式：**TXT / MD**

操作步骤：
1. 点击 **📝 产品描述文档**
2. 在文件选择对话框中选择 `.txt` 或 `.md` 文件
3. 系统读取文件内容后自动发送给 AI pipeline
4. 等同于手动粘贴文本到输入框发送

### 5.2 导入元件清单

支持格式：**JSON / CSV**

用途：导入已有的元件清单，系统自动生成连线图和代码

## 6. 导出功能

点击顶部 **📤 导出** 按钮，出现下拉菜单：

### 6.1 BOM 元件清单（CSV）

- **格式**：嘉立创 JLCPCB 标准 BOM 格式
- **列**：`Comment（型号）, Designator（标号）, Footprint（封装）, Quantity（数量）`
- **用途**：直接上传到嘉立创下单系统

### 6.2 BOM 元件清单（JSON）

- **格式**：完整 JSON，含所有引脚定义
- **用途**：二次开发、数据备份

### 6.3 连线图 PNG

- 导出当前中间画布的截图（2x 分辨率）
- 包含元器件产品图片渲染

### 6.4 KiCad 原理图（SCH）

- **格式**：`.kicad_sch`（KiCad 6 S-expression）
- **用途**：在 KiCad 中打开编辑，或直接上传到嘉立创下单
- 嘉立创支持 KiCad 格式直接下单

### 6.5 全量项目包（ZIP）

包含所有生成内容：
```
project-name.aiprj/
├── project.json          # AIProjectSchema 元数据
├── bom.csv               # BOM 元件清单
├── schematic.kicad_sch   # KiCad 原理图
├── wiring_diagram.png    # 连线图
└── arduino_code.ino      # 生成代码
```

## 7. 编译与烧录

### 7.1 编译面板（右侧窗口 ⚡ 编译 Tab）

**前置条件：**
- 安装 [Arduino CLI](https://github.com/arduino/arduino-cli/releases)
- 将 `arduino-cli.exe` 放入项目 `resources/` 目录（打包前）

**操作步骤：**

1. 在编译 Tab 中，顶部状态显示 `CLI vx.x.x`（绿色=已就绪，红色=未安装）
2. 从下拉菜单选择目标开发板（如 **ESP32-WROOM-32**）
3. （可选）选择连接端口（如果需要上传）
4. 点击按钮：
   - **▶ 编译**：仅编译代码，验证语法
   - **▶ 编译并上传**（选择端口后）：编译并烧录到开发板
   - **■ 停止**：中断正在进行的编译/上传
   - **清空**：清空下方日志

**状态徽章：**
- ✓ 编译成功 / ✗ 编译失败
- ✓ 已上传 / ✗ 上传失败

### 7.2 BOM Tab（元件清单）

自动从当前项目提取数据，显示：

**元件清单表格**
| 型号 | 标号 | 封装 | 数量 |
|------|------|------|------|
| ESP32-WROOM-32D | U1 | ESP32-WROOM-32 | 1 |
| DHT22 | U2 | Sensor_Module | 1 |

**元器件详情卡片**
每个元件显示：
- 标号 + 型号
- 元件类型徽章（如 SENSOR）
- 引脚列表（含 GPIO 编号）

### 7.3 原理图 Tab（引脚连接）

显示当前项目的所有连接：

**引脚连接表**
| 源引脚 | 目标引脚 | 颜色 |
|--------|---------|------|
| ESP32:GPIO21 (SDA) | OLED:SDA | 🔵 蓝 |
| ESP32:GPIO22 (SCL) | OLED:SCL | 🟠 橙 |
| ESP32:GPIO4 | LED:Signal | 🟡 黄 |

## 8. 布线图与面包板视图

中间画布区域支持两种视图切换（**🔗 布线图** / **🔧 面包板**）：

### 8.1 布线图（默认）

- **技术**：ReactFlow 节点连线图
- **节点**：元器件方框 + 文字引脚列表
- **连线**：带颜色和方向的箭头线
- **特性**：可拖拽调整位置，支持缩放/平移，带 MiniMap

### 8.2 面包板视图

- **技术**：SVG 矢量渲染
- **内容**：
  - 芯片/模块以物理引脚排列显示（两侧引脚点）
  - 彩色导线走线（贝塞尔平滑拐角）
  - 左侧电源轨（红+/蓝-）
  - 引脚标注 GPIO 编号
  - VCC 引脚红色、GND 灰色、信号引脚蓝色
- **特性**：
  - 自动检测元器件封装类型
  - 自动网格布局（MCU 居中）
  - 支持 PNG 导出（点击右上角 **Export PNG**）
  - 自动缩放居中

**支持的元器件封装类型：**

| 类型 | 示例 | 引脚分布 |
|------|------|---------|
| ESP32-DevKit | ESP32-WROOM-32 | 双排 38Pin（左侧20+右侧18）|
| Arduino UNO R3 | ATmega328P | 双排 20Pin |
| Arduino Nano | ATmega328P | 双排 30Pin |
| DHT22 | 温湿度传感器 | 单排 4Pin |
| I2C 传感器 | OLED/GY-521 | 单排 4Pin（GND/VCC/SDA/SCL）|
| HC-SR04 | 超声波测距 | 单排 4Pin |
| 继电器模块 | 5V 继电器 | 单排 5Pin |
| PIR 传感器 | HC-SR501 | 3Pin |
| 声音传感器 | 咪头模块 | 4Pin |

## 9. 常见问题

### Q: 编译面板显示 "⚠ CLI 未安装"
**A:** 需要安装 Arduino CLI：
```powershell
winget install ArduinoSA.ArduinoCLI
```
然后将 `arduino-cli.exe` 放入项目 `resources/` 目录。

### Q: 点击发送后 AI 没反应
**A:**
1. 检查右上角模型徽章是否显示"⚠ 无密钥"——需要先配置 API Key
2. 检查网络连接
3. 查看左侧面板 AI 回复区域是否有错误信息

### Q: 导入 PDF 文件失败
**A:** PDF 需要通过拖放方式导入，直接文件选择只能读取文本。推荐将 PDF 内容复制到 TXT 文件后导入。

### Q: 如何查看生成的代码？
**A:** 在右侧编译面板顶部，点击代码区域或查看日志输出。

### Q: 元器件图片不显示
**A:** 这是正常的——元器件图片库是预设的产品图 URL，部分图片可能因网络或图片托管问题无法加载，不影响核心功能。

### Q: 面包板视图和布线图有什么区别？
**A:**
- **布线图**：功能视图，展示元器件之间的逻辑连接关系，适合理解系统架构
- **面包板视图**：物理视图，展示引脚实际物理位置和面包板布线，适合实际焊接/接线参考

### Q: 如何将项目提交到 GitHub？
**A:**
```bash
git add .
git commit -m "feat: 完成 XXX 功能"
git push origin main
```
